# GKE Master Knowledge Base Comprehensive GKE maintenance and policy FAQ. ## GKE Cluster's KCP Upgrades Requirements **Common Question:** I was wondering if there are any requirements for customers to perform upgrades/auto-upgrade for KCP/Node upgrades/restarts (i.e. GKE clusters' KCPs are upgraded at least once a year). If so, are there any exceptions that certain customers may have/how do we identify those customers? Is there any enforcement or is it more of a guideline? **Authoritative Answer:** The documentation suggests that Google Kubernetes Engine (GKE) automatically upgrades cluster control planes (KCPs) on a regular basis for security and stability. This process appears to be mandatory and cannot be disabled, as it is part of GKE's shared responsibility model (GKE Auto-Upgrade FAQ, YAQS: No upgrades in GKE). This policy seems to be enforced, as GKE will forcibly upgrade clusters once their version reaches its End of Life (EOL) date (Document minor version end of life exceptions). According to the documentation, each minor version has a supported lifespan of 14 months, after which it is considered EOL and subject to a forced upgrade (GKE Version Policy). There appear to be ways to temporarily postpone upgrades. One method is by configuring maintenance exclusions, which can prevent automatic upgrades for a specific time period (How to prevent Control Plane upgrade?). For more significant exceptions, a process called "Snowflaking" exists to freeze a control plane's version. However, this is described as a last resort for special cases, requires a high bar for approval, and is not a permanent solution (Preventing auto-upgrades (Control Plane Snowflake), YAQS: Is it possible to have an indefinite pause on GKE upgrades for a cluster?). You can identify a snowflaked cluster by looking for a "Snowflake" status banner on its monitoring dashboard (Check if Snowflake is enabled). --- ## GKE auto-upgrade process for standard and autopilot clusters when there are critical patches **Common Question:** As part of security bulletin notifier and surfacing work our team is doing, could someone on the GKE upgrades team give greater detail on how GKE auto-upgrade versions are determined for both standard and auto-pilot clusters?
