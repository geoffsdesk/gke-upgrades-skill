What This Project Is

This repository contains a purpose-built skill ecosystem that teaches AI coding assistants (Claude and Gemini) how to plan, execute, and troubleshoot GKE cluster upgrades. Instead of relying on generic LLM knowledge, the skills inject domain-specific context — version skew rules, node pool strategies, PDB diagnostics, and runbook templates — directly into the model’s system prompt at inference time.

The project also includes a rigorous evaluation framework that measures whether the skill actually improves output quality, plus a standalone web app for side-by-side comparison and live API testing.

Evaluation & Grading System

Each evaluation consists of a realistic GKE scenario prompt and a set of graded assertions. The same prompt is run twice: once with the skill context injected as a system prompt, and once without (baseline). A grading pass then checks each response against domain-specific expectations.

Current Eval Scenarios (3 evals, 27 total assertions)

  1. Multi-pool Standard upgrade plan — General-purpose + Postgres + GPU pools, version 1.28 → 1.30 (10 assertions)
  2. Autopilot fleet maintenance strategy — 4 clusters across Rapid/Stable channels, EOL notification handling (9 assertions)
  3. Stuck node pool upgrade diagnosis — 3/12 nodes upgraded, 2-hour stall, root cause analysis (8 assertions)

Knowledge Base

The curated knowledge base (data/gke_maint_knowledge.json, v2.0) contains 28 entries sourced from public GKE documentation, with quality scoring to guide skill refinement:

  • Score 2 (14 entries): Verified and recommended — directly referenced in skill materials
  • Score 1 (14 entries): Accepted as accurate — available for future skill expansion
  • All internal/proprietary references have been scrubbed; only public documentation URLs remain

Eval Web App

A standalone, zero-dependency web app (tools/eval-app/) provides:

  • Side-by-side comparison: View with_skill vs without_skill outputs for each eval, with grading overlays
  • Benchmark dashboard: Pass rates, delta analysis, and per-assertion drill-down
  • Live API runs: Enter your Claude or Gemini API key and run any eval prompt live, with or without skill context
  • Runs via python3 tools/eval-app/app.py — Python stdlib only, no pip installs

How to Expand the Eval Set

The eval system is designed for easy extension. To add new evaluations:

  1. Add eval definitions to skill/evals/evals.json. Each eval needs an id, a realistic prompt, and a list of expectations (assertions the grader will check).
  2. Run evals with and without the skill context. Use the eval app’s live run feature or run manually via API. Save outputs as eval-N-output.md in the appropriate workspace/iteration-X/{with,without}_skill/ directory.
  3. Grade outputs by having a model check each output against the expectations. Save as eval-N-grading.json with an array of assertions (expectation + passed + reason).
  4. Update the benchmark by computing aggregate pass rates and saving to benchmark.json. The eval app reads this for dashboard display.

Suggested New Eval Areas

  • Multi-region cluster upgrades with traffic shifting
  • Istio/Anthos Service Mesh version compatibility during upgrades
  • GKE Sandbox (gVisor) node pool upgrades
  • Windows Server node pool upgrades
  • Release channel migration (e.g., Regular → Stable)
  • Webhook/admission controller conflicts during upgrades

How to Expand the Knowledge Base

The knowledge base drives the quality of the skill’s reference materials. To grow it:

  • Add entries to data/gke_maint_knowledge.json following the existing schema (title, content, source_url, tags, quality_score)
  • Score new entries: 2 = directly usable in skill references, 1 = accurate supplementary info, 0 = needs review
  • Run coverage gap analysis by comparing eval assertion failures against knowledge base topics to identify what’s missing
  • Use quality scores to prioritize which entries get incorporated into the skill’s reference .md files

Quick Start

Claude Copy the skill/ directory into your Claude Code project’s .claude/skills/ folder. The skill activates automatically when GKE upgrade topics arise.

Gemini Copy the gemini/ directory into ~/.gemini/extensions/gke-mgmt-lifecycle/. Use slash commands: /gke:check, /gke:plan, /gke:diagnose.

Eval App: python3 tools/eval-app/app.py then open http://localhost:8000 in your browser.
